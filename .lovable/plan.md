

# خطة تحسين أداء التحميل — تقسيم المنصة حسب الدور

## المشكلة
عند فتح التطبيق (سواء على الويب أو PWA)، يتم تحميل **ملف واحد ضخم** يحتوي على تعريفات **260+ صفحة** لكل أنواع المستخدمين (سائق، ناقل، مولّد، مدوّر، إدارة). هذا يعني أن السائق عندما يفتح حسابه، المتصفح يُجهّز مراجع لصفحات الإدارة والناقل والمولّد — وهو لا يحتاجها.

بالإضافة لذلك، الصفحة الرئيسية (Homepage) تُحمّل كاملة حتى لو المستخدم فاتح التطبيق فقط للدخول على حسابه.

## الحل المقترح

### المرحلة 1: تقسيم مسارات Dashboard حسب الدور
بدلاً من ملف واحد ضخم (`DashboardRoutes.tsx` = 555 سطر)، نقسمه إلى ملفات صغيرة:

```text
src/routes/
├── DashboardRoutes.tsx          ← يبقى كـ router رئيسي خفيف
├── dashboard/
│   ├── CommonRoutes.tsx          ← مسارات مشتركة (إعدادات، إشعارات، شات)
│   ├── DriverRoutes.tsx          ← ~15 مسار للسائق فقط
│   ├── TransporterRoutes.tsx     ← مسارات الناقل
│   ├── GeneratorRoutes.tsx       ← مسارات المولّد
│   ├── RecyclerRoutes.tsx        ← مسارات المدوّر
│   ├── AdminRoutes.tsx           ← مسارات الإدارة السيادية
│   └── SpecializedRoutes.tsx     ← disposal, municipal, ERP, etc.
```

**الفكرة**: `DashboardRoutes.tsx` الجديد يقرأ دور المستخدم من `AuthContext` ويُحمّل فقط الملف المناسب:
- سائق → `CommonRoutes` + `DriverRoutes` (15 مسار بدلاً من 260+)
- ناقل → `CommonRoutes` + `TransporterRoutes`
- وهكذا...

### المرحلة 2: إلغاء التحميل المسبق الشامل
في `App.tsx` يوجد سطر يُحمّل كل مسارات Dashboard فوراً:
```typescript
const dashboardRoutesPromise = import('@/routes/DashboardRoutes');
```
سيتم تأخير هذا التحميل ليبدأ فقط عندما يكون المستخدم مُصادق عليه وعلى مسار `/dashboard`.

### المرحلة 3: تخفيف Homepage للمستخدمين المُصادَق عليهم
عندما يكون المستخدم مسجل دخول ويفتح التطبيق، يتم توجيهه مباشرة إلى Dashboard بدون تحميل أقسام الصفحة الرئيسية (Hero, Stats, Partners...).

### المرحلة 4: تحسين PWA Welcome Screen
تقليل الوقت بين فتح التطبيق وظهور المحتوى عبر:
- تخزين حالة المصادقة محلياً لتحديد الوجهة فوراً (dashboard أو homepage)
- عرض skeleton screen أثناء تحميل بيانات المستخدم

## التفاصيل التقنية

### الملفات المتأثرة
| الملف | التغيير |
|---|---|
| `src/routes/DashboardRoutes.tsx` | إعادة هيكلة كاملة — يصبح router ذكي يحمّل حسب الدور |
| `src/routes/dashboard/*.tsx` | 6-7 ملفات جديدة لتقسيم المسارات |
| `src/App.tsx` | إزالة التحميل المسبق الشامل |
| `src/pages/Index.tsx` | إضافة redirect سريع للمستخدمين المُصادَق عليهم |

### النتيجة المتوقعة
- السائق يُحمّل **~15 مسار** بدلاً من **260+**
- حجم JavaScript الأولي ينخفض بنسبة **60-70%** للمستخدم العادي
- وقت فتح التطبيق على PWA ينخفض بشكل ملحوظ
- كل مستخدم يُحمّل فقط ما يخصه
