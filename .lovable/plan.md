## الخطة المرحلية (6 مراحل → 2030)

### المرحلة 1: البنية التحتية والتفريق (الآن)
**الهدف**: كل نوع سائق يرى واجهة مختلفة

| المهمة | التفاصيل |
|--------|---------|
| **تفريق الـ Sidebar** | 3 قوائم جانبية: `company` (الحالي + ربط الجهة)، `hired` (+ العقود + المهام)، `independent` (+ سوق الشحنات + العروض) |
| **تفريق الـ Dashboard** | tabs مختلفة حسب النوع: التابع يرى "المهام + الشحنات"، المؤجر يرى "العقود + المهام"، المستقل يرى "العروض + السوق" |
| **ماتريكس الحجب والعرض** | جدول يحدد ما يظهر/يُحجب لكل نوع (انظر أدناه) |

### المرحلة 2: السائق المستقل — Smart Dispatch (Q3 2026)
**الهدف**: سائق مستقل يتلقى عروض شحنات تلقائياً

| المهمة | التفاصيل |
|--------|---------|
| **Edge Function: smart-dispatch** | خوارزمية توزيع: أقرب سائق متاح ← ضمن نطاق service_area_km ← أعلى تقييم ← أقل رفض |
| **صفحة تسجيل المستقل** | Onboarding مخصص: رخصة + نوع مركبة + نطاق الخدمة + أنواع النفايات المفضلة + تسعيرة |
| **لوحة العروض الواردة** | قائمة بالعروض مع عداد تنازلي (15 دقيقة) + قبول/رفض + تفاصيل الشحنة على الخريطة |
| **ملف السائق العام** | بطاقة تعريفية (rating, trips, acceptance_rate) تظهر لشركات النقل عند البحث |
| **Realtime notifications** | إشعار فوري + صوت عند وصول عرض جديد |

### المرحلة 3: السائق المؤجر — عقود التأجير (Q4 2026)
**الهدف**: شركات النقل تستأجر سائقين أحرار بعقود مؤقتة

| المهمة | التفاصيل |
|--------|---------|
| **واجهة إدارة العقود** | إنشاء/عرض/إنهاء عقود (per_trip, hourly, daily, weekly, monthly) |
| **بحث السائقين للتأجير** | من لوحة الناقل: بحث بالمنطقة + التقييم + التخصص + التوفر |
| **لوحة المؤجر** | العقود النشطة + المهام المسندة + الأرباح المستحقة |
| **نظام الدفع المرحلي** | حساب الأرباح تلقائياً حسب نوع العقد |

### المرحلة 4: التقييم والسمعة والامتثال (Q1 2027)
**الهدف**: منظومة ثقة شاملة

| المهمة | التفاصيل |
|--------|---------|
| **تقييم ثنائي الاتجاه** | الجهة تقيّم السائق + السائق يقيّم الجهة بعد كل مهمة |
| **حساب السمعة التراكمي** | acceptance_rate + rating + total_trips = درجة ثقة |
| **التحقق والتوثيق** | is_verified عبر رفع وثائق (رخصة، هوية، تأمين) + مراجعة إدارية |
| **نظام العقوبات** | رفض متكرر ← تخفيض أولوية ← إيقاف مؤقت ← حظر |
| **لوحة امتثال السائق** | انتهاء الرخصة + التأمين + الفحص الدوري |

### المرحلة 5: الذكاء والتحليلات (2028)
**الهدف**: تحسين الكفاءة بالبيانات

| المهمة | التفاصيل |
|--------|---------|
| **تحليلات الأداء** | Dashboard تحليلي: معدل القبول، متوسط وقت الاستجابة، المسافات، الأرباح |
| **توقع الطلب بالـ AI** | تحليل أنماط الشحنات ← اقتراح مناطق ذات طلب عالٍ للسائق المستقل |
| **تحسين المسارات الجماعي** | تجميع شحنات متعددة في رحلة واحدة (Batch Routing) |
| **تقارير الجهات** | لكل جهة ناقلة: تقرير أداء سائقيها (تابعين + مؤجرين) |

### المرحلة 6: التوسع والتكامل (2029-2030)
**الهدف**: منظومة كاملة جاهزة للسوق

| المهمة | التفاصيل |
|--------|---------|
| **سوق الشحنات المفتوح** | السائقون المستقلون يتصفحون شحنات متاحة ويقدمون عروض أسعار |
| **نظام المزايدة** | عكس المزاد: الشحنة تُطرح ← السائقون يتنافسون بالسعر الأقل |
| **تأمين الشحنات** | ربط مع شركات تأمين لتغطية المخاطر |
| **محفظة إلكترونية حقيقية** | ربط مع بوابة دفع لتسوية الأرباح |
| **تطبيق السائق المخصص** | PWA مُحسّن للموبايل بوضع offline-first |
| **API خارجي** | فتح واجهة برمجية لربط أنظمة الجهات الخارجية |

## التفاصيل التقنية

### 1. تفريق الـ Sidebar (المرحلة 1)
- تعديل `DashboardLayout.tsx` ← استخدام `useDriverType()` لتحديد `driverMenuItems` حسب النوع
- 3 مصفوفات قوائم: `companyDriverMenu`, `hiredDriverMenu`, `independentDriverMenu`

### 2. تفريق الـ Dashboard (المرحلة 1)
- تعديل `DriverDashboard.tsx` ← عرض tabs مختلفة حسب `driverType`
- التابع: المهام | الشحنات | أدوات الميدان | المالية | حسابي
- المؤجر: العقود | المهام | أدوات الميدان | المالية | حسابي
- المستقل: العروض | السوق | أدوات الميدان | المالية | حسابي

### 3. Smart Dispatch Engine (المرحلة 2)
- Edge Function تُستدعى عند إنشاء شحنة بدون سائق
- استعلام: سائقون مستقلون + متاحون + ضمن النطاق + مرتبون بالتقييم
- إنشاء سجل في `driver_mission_offers` مع `expires_at = now() + 15min`
- Cron job لتنظيف العروض المنتهية
