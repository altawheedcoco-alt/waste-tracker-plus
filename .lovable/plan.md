

# خطة: إعادة هيكلة القائمة الجانبية بنظام هرمي (صفحة → مجموعة → عناصر)

## الفكرة

تحويل القائمة الجانبية من مستوى واحد (مجموعات مسطحة) إلى نظام هرمي من 3 مستويات:

```text
📄 اسم الصفحة / القسم  ← عنوان بارز (Section Header)
  📁 اسم المجموعة      ← قابل للطي (Group)
    📌 العنصر / التبويب  ← رابط مباشر (Item)
```

## الوضع الحالي

القائمة تعرض ~50 مجموعة متتالية بدون تقسيم واضح. `CATEGORY_ORDER` يرتبها لكن بدون عناوين أقسام مرئية.

## ما سيتم تنفيذه

### 1. إضافة مفهوم "القسم" (Section) في الـ Config
- إضافة `SidebarSectionConfig` interface يحتوي `id`, `labelAr`, `labelEn`, `icon`, `groupIds[]`
- تعريف الـ 14 قسم مع ربط كل مجموعة بقسمها

### 2. تطوير `SidebarNavGroup` لدعم عناوين الأقسام
- إضافة مكون `SidebarSectionHeader` يعرض اسم القسم كعنوان بارز (غير قابل للنقر، فقط تصنيف بصري)
- تصميم بخط أصغر، لون خافت، مع خط فاصل - يشبه section headers في تطبيقات الموبايل

### 3. تعديل `DashboardLayout.tsx` لتجميع المجموعات تحت أقسامها
- بدلاً من عرض المجموعات مسطحة، تُجمع تحت section headers
