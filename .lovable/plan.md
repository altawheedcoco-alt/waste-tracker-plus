# خطة إضافة قسم Reels (ريلز) للمنصة

## الفكرة باختصار 

هناك بس اضافه وتعديل مهم جدا في الريلز اللي بيشوف الريلز الجهه المولده الجهه الناقله جهه التدوير جهه التخلص النهائي وجميع تابعيهم يرون الريلز في العام انما الجهات المولده ترى الريلز الخاصه بالجهات المولده مثلها وترى الريلز الخاص بالجهات المشتركه معها وكذلك الجهات المشتركه معها ترى الريلز الخاص بها وكذلك السائقين المستقلين يرون الريلز الخاص بي بسائقين الاخرين المستقلين واذا كان السائق قد تم ربطه بجهه شريكه او مرتبطه معه ولكنه ايضا مستقل او سائق مستقل فانه يرى الريز الخاص بالجهه المرتبط معها والعكس صحيح

إضافة قسم **Reels** مستقل عن الحالات (Stories)، يشبه ريلز إنستجرام/تيك توك — فيديوهات قصيرة عمودية يتصفحها المستخدم بالسحب لأعلى/أسفل، مع تفاعلات كاملة.

## الفرق بين الحالات والريلز

```text
┌──────────────────┬──────────────────────────┬──────────────────────────┐
│                  │ الحالات (Stories)         │ الريلز (Reels)           │
├──────────────────┼──────────────────────────┼──────────────────────────┤
│ المدة            │ تختفي بعد 24 ساعة        │ دائمة (لا تنتهي)         │
│ التصفح           │ أفقي بين المستخدمين      │ عمودي (سحب لأعلى/أسفل)  │
│ المحتوى          │ صور + فيديو + نص         │ فيديو فقط (15-60 ثانية)  │
│ التفاعل          │ ردود خاصة               │ لايك + تعليقات عامة      │
│ الاكتشاف         │ شركاء فقط               │ كل المنصة (Explore)      │
└──────────────────┴──────────────────────────┴──────────────────────────┘
```

## هل هي زيادة على الفاضي؟

**لا، لأنها تحل مشكلة مختلفة:**

- الحالات = تواصل مؤقت بين الشركاء (مثل واتساب)
- الريلز = محتوى دائم للتوعية والتسويق البيئي يصل لكل مستخدمي المنصة

## خطة التنفيذ

### 1. قاعدة البيانات

- جدول `reels` جديد (user_id, organization_id, video_url, caption, hashtags, duration, view_count, like_count, is_active)
- جدول `reel_likes` و `reel_comments` للتفاعلات
- RLS policies مبنية على نفس قواعد المنشورات والحالات

### 2. المكونات الجديدة

- `**ReelsFeed.tsx**` — عارض عمودي بملء الشاشة (Fullscreen vertical swiper) مع تشغيل تلقائي للفيديو
- `**ReelItem.tsx**` — الريل الواحد مع شريط جانبي TikTok-style (لايك، تعليق، مشاركة، حفظ)
- `**ReelUploadDialog.tsx**` — رفع فيديو مع قص (trim) واختيار صورة غلاف
- `**ReelComments.tsx**` — تعليقات Bottom Sheet

### 3. الصفحات

- صفحة `/reels` أو `/dashboard/reels` كتبويب جديد
- إضافة أيقونة Reels في الشريط السفلي/القائمة الجانبية

### 4. التكامل مع النظام الحالي

- نفس نظام الصلاحيات والإعدادات (platform_settings)
- إشعارات عند التفاعل (لايك/تعليق) عبر نظام الإشعارات الموجود
- نفس منظومة الرفع الذكي (chunked upload + compression)

## التفاصيل التقنية

- استخدام `Intersection Observer` لتشغيل/إيقاف الفيديو تلقائياً عند الظهور
- `framer-motion` للسحب العمودي مع `snap` بين الريلز
- دعم الـ hashtags للتصنيف والبحث
- خوارزمية ترتيب بسيطة (الأحدث + الأكثر تفاعلاً)
