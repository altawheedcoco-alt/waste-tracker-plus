# خطة: تطوير مركز عمليات الدردشة حسب نوع الجهة (Org-Aware Chat Operations)

## المشكلة الحالية 

ترك الأساسيات وضف عليها الاتي

النظام يعامل كل الجهات بنفس الطريقة - نفس الأوامر `/slash`، نفس الأزرار على البطاقات، نفس خيارات `ChatActionPanel`. لكن في الواقع:

- **المولد**: يحتاج طلب جمع، تصنيف مخلفات، جدولة، متابعة شهادات تدوير
- **الناقل**: يحتاج تعيين سائق، تتبع أسطول، تغيير حالة شحنة، فحص مركبة
- **المدوّر**: يحتاج استقبال شحنات، تأكيد أوزان، إصدار شهادات تدوير، رصد جودة
- **التخلص**: يحتاج تأكيد استقبال، إصدار شهادة تخلص آمن، رصد بيئي

---

## ما سيتم تنفيذه

### 1. سجل أوامر Slash ذكي حسب نوع الجهة

إنشاء `src/config/chatOrgCommands.ts` - خريطة أوامر مخصصة لكل جهة:

**المولد**:

- `/collect` - طلب جمع مخلفات (جديد)
- `/classify` - تصنيف نوع النفايات (جديد)
- `/schedule` - جدولة موعد الجمع (جديد)
- `/work-order` - إنشاء أمر شغل رقمي (جديد)
- `/cert` - طلب شهادة تدوير (جديد)

**الناقل**:

- `/assign-driver` - تعيين سائق لشحنة (جديد)
- `/fleet-check` - فحص سلامة مركبة (جديد)
- `/route` - تحسين مسار (جديد)
- `/fuel` - تسجيل وقود (جديد)
- `/handover` - تسليم شحنة (جديد)

**المدوّر**:

- `/receive` - تأكيد استلام شحنة (جديد)
- `/weigh` - تسجيل وزن فعلي (جديد)
- `/quality` - فحص جودة المواد (جديد)
- `/recycle-cert` - إصدار شهادة تدوير (جديد)
- `/reject` - رفض شحنة مع السبب (جديد)

**التخلص النهائي**:

- `/dispose` - تأكيد التخلص الآمن (جديد)
- `/monitor
