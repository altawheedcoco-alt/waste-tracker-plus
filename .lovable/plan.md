

# خطة تطوير الدردشة كـ "همزة وصل" تشغيلية شاملة

## الرؤية
تحويل الدردشة من مجرد أداة تواصل نصي إلى **مركز عمليات تفاعلي** يربط بين قواعد بيانات الطرفين — يستطيع المستخدم من داخل المحادثة: طلب توقيع، إرفاق فاتورة/شحنة/مستند، البحث في أرشيفه، وتنفيذ إجراءات سريعة دون مغادرة نافذة الدردشة.

---

## المحور الأول: نظام الأوامر السريعة (Slash Commands) داخل الدردشة

### الفكرة
عند كتابة `/` في حقل الإدخال تظهر قائمة أوامر تفاعلية:

```text
/sign     → طلب توقيع على مستند
/invoice  → إرسال/طلب فاتورة
/shipment → مشاركة بطاقة شحنة
/doc      → إرفاق مستند من الأرشيف
/stamp    → طلب ختم رسمي
/track    → مشاركة رابط تتبع شحنة
```

### التنفيذ
- إنشاء `src/components/chat/SlashCommandMenu.tsx` — قائمة منسدلة تظهر عند `/`
- إنشاء `src/config/chatSlashCommands.ts` — سجل الأوامر المتاحة حسب نوع الجهة
- تعديل `EnhancedChatInput.tsx` لكشف `/` وعرض القائمة (مثل منطق `@mention` الموجود)

---

## المحور الثاني: بطاقات تفاعلية داخل الرسائل (Rich Message Cards)

### الفكرة
عند تنفيذ أمر أو إرفاق عنصر، يظهر في المحادثة كـ **بطاقة تفاعلية** (وليس رابط نصي):

- **بطاقة شحنة**: رقم الشحنة + الحالة + نوع المخلف + زر "فتح التفاصيل"
- **بطاقة فاتورة**: رقم الفاتورة + المبلغ + الحالة + زر "دفع/معاينة"
- **بطاقة طلب توقيع**: اسم المستند + زر "وقّع الآن" مباشرة
- **بطاقة مستند**: اسم الملف + نوعه + زر "تحميل/معاينة"

### التنفيذ
- إنشاء `src/components/chat/cards/` مجلد يضم:
  - `ShipmentCard.tsx` — بطاقة شحنة مصغرة
  - `InvoiceCard.tsx` — بطاقة فاتورة مصغرة
  - `SigningRequestCard.tsx` — بطاقة طلب توقيع مع زر "وقّع"
  - `DocumentCard.tsx` — بطاقة مستند مع معاينة
- إنشاء `ChatMessageCardRenderer.tsx` — يكتشف نوع الرسالة ويعرض البطاقة المناسبة
- تعديل `EnhancedChatMessages.tsx` و `EncryptedChatWidget.tsx` لعرض البطاقات

## المحور الثالث: البحث السياقي من داخل الدردشة

### الفكرة
زر بحث داخل حقل الإدخال يتيح البحث في:
- شحناتي → اختيار شحنة → إرسالها كبطاقة
- فواتيري → اختيار فاتورة → إرسالها
- مستنداتي → اختيار مستند → إرفاقه
- طلبات التوقيع → اختيار طلب → مشاركته

### التنفيذ
- إنشاء `src/components/chat/ChatResourcePicker.tsx` — نافذة بحث مع تبويبات (شحنات، فواتير، مستندات)
- استخدام الـ Repositories الموجودة (`InvoicesRepository`, `ShipmentsRepository`, إلخ) لجلب البيانات
- عند الاختيار: إرسال رسالة من نوع خاص `message_type: 'resource_card'` مع `metadata`

---

## المحور الرابع: إجراءات فورية داخل المحادثة (In-Chat Actions)

### الفكرة
المستلم يستطيع تنفيذ إجراءات مباشرة من بطاقة الرسالة:
- **"وقّع الآن"** على بطاقة طلب التوقيع → يفتح نافذة التوقيع → يُرسل التوقيع كرد تلقائي
- **"اعتمد الفاتورة"** → تغيير حالة الفاتورة → إشعار تلقائي
- **"قبول الشحنة"** → تحديث حالة الشحنة
- **"رفض مع سبب"** → إدخال سبب الرفض

### التنفيذ
- تعديل بطاقات المحور الثاني لإضافة أزرار إجراء
- ربط الأزرار بـ Repositories و Hooks الموجودة (`useMentionSigning`, `useSigningInbox`)
- إرسال رسالة نظام تلقائية عند تنفيذ الإجراء (مثل: "✅ تم التوقيع على الفاتورة #INV-001")

## المحور الخامس: نفس المنطق في الملاحظات (Notes)

### الفكرة
تطبيق نظام Slash Commands والبطاقات التفاعلية في `NotesTab.tsx` و `AddNoteDialog.tsx`:
- الملاحظة يمكن أن تحتوي على إشارة لشحنة أو فاتورة أو مستند
- تظهر كبطاقات تفاعلية داخل الملاحظة
- يمكن للطرف الآخر التفاعل معها
