
# خطة تحسين التتبع الحي للشحنات مع خرائط تفاعلية

## ملخص المشروع
تطوير نظام التتبع الحي ليعرض خريطة تفاعلية داخل كل شحنة مباشرة (Inline Map) دون الحاجة لفتح نافذة منفصلة، مع تحسين عرض مسار السائق الفعلي ومقارنته بالمسار المخطط.

---

## الحالة الراهنة للنظام

يمتلك النظام حالياً مكونات تتبع متقدمة:
- **UnifiedShipmentTracker**: يعرض خريطة مصغرة داخل صفحة الشحنة
- **LiveTrackingMapDialog**: نافذة منفصلة للتتبع المباشر تفتح عند الضغط على زر
- **GoogleMapsShipmentTracking**: يعرض نقاط الاستلام والتسليم وموقع السائق
- **GoogleLiveTrackingMap**: يرسم المسار الفعلي للسائق في الوقت الحقيقي

---

## الهيكل البرمجي

### واجهات البيانات الجديدة

```typescript
interface InlineMapProps {
  shipmentId: string;
  pickupAddress: string;
  deliveryAddress: string;
  driverId?: string | null;
  status: string;
  collapsible?: boolean;  // إمكانية الطي
  defaultExpanded?: boolean;  // مفتوح افتراضياً
  height?: number;  // ارتفاع الخريطة
}

interface RouteVisualizationData {
  plannedRoute: [number, number][];  // المسار المخطط
  actualRoute: [number, number][];   // المسار الفعلي
  driverLocation: { lat: number; lng: number } | null;
  progress: number;  // نسبة الإنجاز (0-100)
  remainingDistance: number;  // كم
  estimatedArrival: Date | null;
}
```

## اشتراكات الوقت الحقيقي (Realtime)

سيتم استخدام اشتراكات Supabase للتحديث اللحظي:

```typescript
// في كل مكون خريطة
const channel = supabase
  .channel(`shipment-map-${shipmentId}`)
  .on('postgres_changes', {
    event: 'INSERT',
    schema: 'public',
    table: 'driver_location_logs',
    filter: `driver_id=eq.${driverId}`
  }, (payload) => {
    // تحديث موقع السائق
    updateDriverPosition(payload.new);
  })
  .subscribe();
```
