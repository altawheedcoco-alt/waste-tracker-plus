

# خطة تطوير منظومة المقاول البلدي — دراسة علمية وتطبيقية

## نتائج البحث والدراسة

بناءً على دراسة القانون المصري 202/2020 ولائحته التنفيذية، وأفضل الممارسات العالمية في إدارة مخلفات البلدية، ومقارنة مع الأنظمة الحالية المطبقة في القاهرة الكبرى:

### الفجوات المكتشفة في النظام الحالي

| الوحدة | الموجود | المفقود |
|---|---|---|
| العقود الحكومية | مرجع عقد نصي فقط | عقود كاملة + SLA + غرامات تعاقدية + مؤشرات أداء |
| إثبات الأداء | زر "إنهاء رحلة" فقط | صور قبل/بعد + GPS timestamp + توقيع مشرف |
| التقارير الدورية | لا يوجد | تقارير يومية/أسبوعية/شهرية للجهة الحكومية |
| الجزاءات والغرامات | جدول فارغ | نظام كامل بأنواع المخالفات وآليات الاستقطاع |
| السلامة المهنية | لا يوجد | تسجيل حوادث وإصابات العمال + معدات الوقاية |
| جرد المعدات اليدوية | ملخص نصي | سجل تفصيلي لاستلام وصرف العُهد (مقشات، أكياس، قفازات) |
| خريطة العمليات | لا يوجد | خريطة حية لمواقع الطواقم والصناديق |
| تقييم أداء الطواقم | لا يوجد | KPIs لكل طاقم (معدل إنجاز، شكاوى، غياب) |

---

## خطة التطوير

### 1. نظام العقود الحكومية ومؤشرات الأداء (SLA)
**جدول جديد: `municipal_contracts`**
- بيانات العقد (رقم، تاريخ بدء/انتهاء، قيمة شهرية/سنوية)
- الجهة المتعاقدة (محافظة/هيئة نظافة/حي)
- مؤشرات الأداء التعاقدية (KPIs): نسبة تغطية، حد أقصى للشكاوى، حد أدنى لأطنان الجمع
- حالة العقد (نشط/منتهي/موقوف)

**صفحة: `MunicipalContracts.tsx`**
- عرض العقود + مؤشرات الأداء مقابل المستهدف
- تنبيهات تلقائية قبل انتهاء العقد

### 2. تطوير نظام الغرامات والجزاءات
**تحديث `contract_penalties`** (الجدول موجود لكن الصفحة غير مفعلة)

**صفحة: `PenaltiesManagement.tsx`**
- أنواع المخالفات المصرية الشائعة: عدم رفع صندوق، تأخر عن الموعد، ترك مخلفات متناثرة، عدم غسل الصندوق
- ربط الغرامة بالشكوى/المنطقة/الطاقم
- حساب إجمالي الغرامات الشهرية وخصمها من مستحقات العقد

### 3. نظام إثبات الأداء الميداني (Proof of Service)
**جدول جديد: `proof_of_service`**
- صور قبل/بعد لكل نقطة جمع
- GPS + timestamp تلقائي
- توقيع المشرف الميداني
- ربط بالرحلة والمنطقة

**تطوير صفحة `CollectionTrips.tsx`**
- إضافة زر "إثبات أداء" لكل رحلة
- عرض الصور والموقع في كارت الرحلة

### 4. نظام حوادث وسلامة العمال
**جدول جديد: `worker_incidents`**
- نوع الحادث (إصابة عمل، حادث مروري، تعرض لمواد خطرة)
- العامل المصاب + الطاقم + المنطقة
- إجراءات متخذة + حالة المتابعة

**صفحة: `WorkerSafety.tsx`**
- تسجيل الحوادث + إحصائيات السلامة
- تنبيه عند تكرار الحوادث بمنطقة معينة

### 5. نظام العُهد والمستلزمات (Inventory Custody)
**جدول جديد: `equipment_custody`**
- صرف واستلام المستلزمات (مقشات، أكياس، قفازات، زي موحد، عربات يد)
- تتبع الكميات لكل عامل/طاقم
- تنبيه نفاد المخزون

**صفحة: `EquipmentCustody.tsx`**
- سجل الصرف اليومي + رصيد المخزون

### 6. التقارير الدورية للجهة الحكومية
**صفحة: `MunicipalReports.tsx`**
- التقرير اليومي: رحلات مكتملة، أطنان مجموعة، حضور العمال
- التقرير الأسبوعي: مقارنة أداء المناطق، شكاوى محلولة
- التقرير الشهري: إجمالي الأداء مقابل SLA + غرامات + تكاليف

### 7. تطوير لوحة المراقبة الرئيسية
**تحديث `MunicipalDashboard.tsx`**
- إضافة شريط أداء العقد (SLA compliance %)
- إضافة مؤشر السلامة (أيام بدون حوادث)
- إضافة ملخص الغرامات الشهرية
- تحسين UX مع tabs للتنقل بين الأقسام
