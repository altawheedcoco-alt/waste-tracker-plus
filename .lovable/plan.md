

# خطة تطوير: الجدولة اليدوية + قسم منشورات المنصة

## الوضع الحالي

### 1. الجدولة اليدوية — غير موجودة ❌
نموذج إضافة الخبر عند المدير يحتوي فقط على زر "نشر مباشرة" (Switch). عند تفعيله يتم تعيين `published_at = now()`. **لا يوجد حقل لاختيار تاريخ ووقت نشر مستقبلي**، مما يعني أن المدير لا يستطيع جدولة خبر يدوياً ليظهر في وقت معين.

### 2. منشورات المنصة على الصفحة الرئيسية — غير موجودة ❌
لا يوجد قسم في الصفحة الرئيسية يعرض "منشورات المنصة" (مثل مدونة أو إعلانات رسمية). الموجود فقط شريط أخبار (NewsTicker) وصفحة أخبار `/news`.

## التطوير المطلوب

### 1. إضافة الجدولة اليدوية لنموذج الأخبار
**ملف: `AdminNewsManager.tsx`**
- إضافة حقل "تاريخ ووقت النشر المجدول" (datetime-local input)
- عند اختيار تاريخ مستقبلي: يُحفظ الخبر بـ `is_published = false` و `published_at = التاريخ المختار`
- الـ Cron Job الموجود أصلاً سينشره تلقائياً عندما يحين الموعد
- عند اختيار "نشر مباشرة": يُنشر فوراً كالسابق
- إضافة مؤشر بصري "مجدول" بجانب الأخبار المجدولة مع عرض الموعد

### 2. إنشاء قسم "منشورات المنصة" على الصفحة الرئيسية
**جدول جديد: `platform_posts`**
- عنوان، محتوى، صورة غلاف، تصنيف، كاتب (اسم)، تاريخ النشر
- `is_published`, `published_at`

**صفحة جديدة: `src/pages/PlatformPosts.tsx`**
- عرض جميع منشورات المنصة بتصميم بطاقات جذاب

**قسم في الصفحة الرئيسية (Index.tsx)**
- إضافة قسم "منشورات المنصة" مع آخر 3 منشورات + زر "عرض الكل"

**إدارة المنشورات عند المدير: `AdminPlatformPosts.tsx`**
- إضافة/تعديل/حذف المنشورات مع دعم الجدولة
