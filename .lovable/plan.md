## خطة التطوير الرأسي — مرتبة بالأولوية

### المرحلة 9: المولّد (Generator) — الأكثر استخداماً
- **فحص لوحة التحكم**: مراجعة كل ودجت وتاب وزرار — هل المولّد يوصل لكل اللي محتاجه بسرعة؟
- **تجربة إنشاء الشحنة**: هل سهلة؟ هل فيها اختصارات؟ هل ممكن نضيف "شحنة سريعة" بنقرة واحدة؟
- **متابعة الشحنات**: عرض مبسط لحالة كل شحنة + إشعارات لحظية
- **سجل المخلفات**: تسهيل التسجيل اليومي + تقارير شهرية تلقائية
- **ربط الشركاء**: تسهيل اكتشاف الناقلين والمدورين وربطهم

### المرحلة 10: الناقل (Transporter) — العمود الفقري
- **غرفة العمليات**: مراجعة شاملة لكفاءة إدارة الشحنات والسائقين
- **إدارة الأسطول**: تحسين تتبع المركبات + الصيانة + الوقود
- **السائقين**: تعيين تلقائي + تقييم أداء + ورديات
- **المحاسبة**: ربط الفوترة بالتسليم + تسوية حسابات الشركاء
- **التقارير التشغيلية**: تقارير يومية/أسبوعية تلقائية

### المرحلة 11: المدوّر (Recycler) ✅ الدفعة الأولى
- [x] RecyclerIncomingQuality — فحص جودة الواردات (شحنات بانتظار القبول/الرفض)
- [x] RecyclerProductionOutput — مخرجات التدوير (وارد/مُعاد/معدل تحويل/توزيع بالنوع)
- [x] RecyclerQuickFAB — زر عائم (شحنة/فحص/وزن/شهادة/تقارير)
- [x] دمج الودجات في لوحة التحكم بتخطيط grid ثنائي
- **متبقي**: تحسين إدارة الحاويات + تقارير الإنتاج التفصيلية

### المرحلة 12: التخلص النهائي (Disposal) ✅ الدفعة الأولى
- [x] DisposalSiteStatus — حالة مواقع الدفن (سعة/امتلاء/تراخيص)
- [x] DisposalEnvironmentalMonitor — رصد بيئي (خطرة/سلامة/طرق تخلص)
- [x] DisposalQuickFAB — زر عائم (عملية/شحنة/وزن/شهادة/تقارير)
- [x] دمج الودجات في لوحة التحكم بتخطيط grid ثنائي

### المرحلة 13: السائق (Driver) — مستقل وتابع لشركة
- **تجربة الجوال**: السائق يستخدم الموبايل 100% — كل زرار لازم يكون واضح وكبير
- **السائق المستقل**: رادار الطلبات + العروض + الأرباح
- **سائق الشركة**: المهام اليومية + الأداء + التواصل مع الإدارة

### المرحلة 14: الاستشاري والمكتب الاستشاري
- **إدارة العملاء**: مراجعة ملفات العملاء + تعيينات + صلاحيات الوصول
- **إصدار المستندات**: تسهيل إصدار التقارير والشهادات البيئية
- **المهام والمتابعة**: نظام مهام مرتبط بكل عميل

### المرحلة 15: مكتب النقل + جهة ISO + الرقابة + مقاول البلديات
- **مكتب النقل**: إدارة الأساطيل المتعددة + الاعتمادات الفنية
- **ISO**: تحسين التحقق من المستندات + منح الشهادات
- **الرقابة**: لوحة إشراف شاملة
- **مقاول البلديات**: إن لم تكن موجودة — بناء لوحة تحكم كاملة

### المرحلة 16: الموظف + مدير النظام
- **الموظف**: تحسين تجربة الموظف المحدود الصلاحيات
- **الأدمن**: مراجعة مركز القيادة والسيطرة

## طريقة العمل

كل مرحلة هبدأها بـ:
1. **فحص شامل** لكل ملف متعلق بالجهة (لوحة تحكم، مسارات، مكونات، hooks)
2. **تحديد الفجوات** — إيه ناقص وإيه محتاج تحسين
3. **تنفيذ التعديلات** مباشرة بدون انتظار

**مبدأ "المرونة أولاً"** ثابت — كل إضافة اختيارية.

هبدأ بالمرحلة 9 (المولّد) فوراً لما تقولي "أكمل".
