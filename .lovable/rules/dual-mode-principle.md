# Memory: architecture/dual-mode-principle
Updated: now

## المبدأ المعماري الأساسي: التلقائي أولاً (Auto-First)

كل ميزة أو إجراء في النظام يجب أن يُبنى على أساس مزدوج:

1. **الوضع التلقائي (افتراضي)**: يعمل بدون تدخل المستخدم، يوفر الوقت والجهد
2. **الوضع اليدوي (اختياري)**: متاح دائماً كبديل للتحكم الكامل

### أمثلة التطبيق:
- إنشاء شهادات الاستلام → تلقائي عند تسليم الشحنة + يدوي من زر الإنشاء
- تغيير حالات الشحنة → تلقائي عبر GPS/Geofencing + يدوي من واجهة الحالات
- حساب الأسعار → تلقائي من خطاب الترسية + يدوي بإدخال السعر
- الإشعارات → تلقائية عند الأحداث + يدوية بالإرسال المباشر
- التقارير → تلقائية دورية + يدوية عند الطلب

### القاعدة: الأتمتة هي الوضع الافتراضي، واليدوي هو الاستثناء.
